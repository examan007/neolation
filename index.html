Design

Architectural patterns are important; the most important being MVC or model-view-controller. Modern web applications should adhere to this pattern.

Concerns should be separated as interchangeable components The different concerns are data model, layout and style view, and event handling and update control code.

Good reason to adhere to this pattern is simplification and reuse of components. Separation of concerns eliminates unnecessary coding of duplicate functionality decreasing maintenance cost and increasing feature enhancements.

Implement

Javascript is the de-facto web application language; that is Javascript is all that is needed to build a modern end-to-end web application.

Given first class functions in Javascript facilitates the closure; this means Javascript supports functional programming the third computing paradigm. The second paradigm is object oriented programming. The first and most understood is procedural programming.

And of course Agile; this is how code is developed naturally but needing more people than just a coder in order to scale to meet high production demands.

Test

Test. Test. Test. Yes, but how is testing more effective? This comes back to design. Well separated concerns, good functional method, and well defined interface lend to test programs exercising all the code.

Well designed modules need less testing because components may be extended with new functionality and not make any changes to the existing and working components.

New feature functionality may be developed independent of the working system and integrated to the working system without impacting existing relied upon behaviour.
